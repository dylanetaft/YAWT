Primary user-facing QUIC API for connection management and I/O.

Typedef Documentation

◆ YAWT_Q_Connection_t

typedef struct YAWT_Q_Connection YAWT_Q_Connection_t

A QUIC connection.

Note
Owned by the QUIC layer (created in con_create, destroyed in con_free). Indexed in global CID hash tables; found via con_find_by_cid. Threading: NOT thread-safe — one libev loop drives all connections. All fields are mutated only on that thread, synchronously with con_rx / con_maintain. Most fields are QUIC-internal; user_data is the app's.

◆ YAWT_q_con_maintain()

void YAWT_q_con_maintain ( double now)

Unified maintenance pass over all connections.

Parameters
nowCurrent time (e.g., from ev_now()).
Note
Retransmits lost frames, enforces idle timeouts, sends keepalive PINGs, and flushes queued packets (emits EVT_TX). May free connections whose close/idle has expired, each emitting EVT_CLOSE via YAWT_q_con_free. Call periodically from the event loop.

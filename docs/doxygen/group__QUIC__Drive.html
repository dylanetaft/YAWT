Detailed Description

Functions the user must call from their event loop to drive the QUIC stack.

The QUIC library is not self-running. The user owns the event loop and must call these two functions each iteration:

  • YAWT_q_con_rx() on incoming UDP data
  • YAWT_q_con_maintain() on a timer (see YAWT_q_con_get_maint_config())

void YAWT_q_con_maintain ( double now)

Maintenance function for quic.

Userspace is responsible for calling this periodically on a frequency defined by YAWT_q_con_get_maint_config()->min_maint_interval see examples for usage

Parameters
nowCurrent time (e.g., from ev_now()).
Note
Retransmits lost frames, enforces idle timeouts, sends keepalive PINGs, and flushes queued packets (emits EVT_TX). May free connections whose close/idle has expired, each emitting EVT_CLOSE via YAWT_q_con_free.

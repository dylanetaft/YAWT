Detailed Description

Per-stream H3 parse state.

Note
Lives in a preallocated slot pool on the H3 connection (slot index is NOT the stream id — ids are sparse and grow unbounded, so we store stream_id and linear-scan, like QUIC's stream_meta). A slot is claimed (in_use=true) when assigned to a stream id. A stream outlives any single frame and carries many of them, so the current frame is a union member, reset when advancing to the next frame.
